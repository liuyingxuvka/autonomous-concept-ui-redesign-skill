# README Hero Design Note

## Project summary

This Codex skill runs an autonomous concept-led UI redesign pipeline: product
inspection, functional framing, concept search, implementation, iteration,
deviation review, app icon realization, and screenshot/geometry QA.

## Target users

Codex agents and FlowPilot routes working on substantial product UI redesigns
where a grounded visual target and rendered QA matter.

## Core problem

Large UI redesigns can drift into decorative concepts, feature bloat, weak app
identity, or unverified layouts that fail once rendered with real content.

## Core workflow

Inspect the current product, frame the functional UI, search candidate concepts,
select a grounded target, implement within the real architecture, iterate from
screenshots, and close with geometry, icon, and visual QA evidence.

## Hero tagline

A standalone Codex skill for concept-led UI redesign, implementation, iteration, and visual QA.

## Visual concept

A bright redesign studio sculpture where an existing interface separates into
functional slices, passes through a concept lens, reassembles into
implementation blocks, and returns through screenshot QA and visual diff
feedback.

## Material language

Satin enamel interface slabs, magnetic component blocks, color-calibration
ribbons, polished aluminum rails, and soft fabric pinboard texture.

## Image keywords

premium workflow visual, product inspection, concept lens, implementation
components, screenshot QA, visual diff loop

## Generation method

Reused the previously generated concept hero for the concept-led UI redesign
workflow because this standalone skill now includes that workflow internally.
The prompt used enamel, magnetic, calibration, aluminum, and fabric studio
materials to distinguish the UI redesign workflow from repository or memory
system visuals.
